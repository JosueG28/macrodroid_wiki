    Action Blocks

    Description

    Action Blocks let you create reusable groups of actions that can be triggered as a single step from any macro. They help simplify complex workflows and prevent repeating actions across macros.

    When to Use

    Action Blocks are ideal for minimizing duplicated actions across macros. For example, you might set screen brightness low and volume to zero in one macro (triggered by time of day) and another (triggered by location, like at the cinema). By grouping these into an Action Block, you call it once per macro. If you tweak the settings later, update just the blockâ€”not every macro.

    Where can I find the Action Blocks?

    You can access Action Blocks from the "Action Blocks" tile on the home screen.
    Tap the "+" button to create a new block.

    Each Action Block must have:
    - A unique name
    - At least one action

    Optional:
    - Description
    - Category (helps organize blocks in the list)
    - Input parameters
    - Output parameters

    How to Create an Action Block

    1. Open MacroDroid > Action Blocks.
    2. Tap the â€œ+â€ button.
    3. Add actions.
    4. Save and name it.

    Input Parameters

    Inputs make Action Blocks dynamicâ€”like passing data into the block so it can work with different values each time. Theyâ€™re optional, but extremely useful for reusability.

    Inputs: Values passed from the macro to the Action Block.

    Each input has a default value, used during local tests or if no value is passed from the macro.

    For example, if your macro has a local variable named price with a value of 150, you can pass it into the Action Block using Magic Text:

    {lv=price} â†’ represents the current value of the local variable price (in this case, 150).

    You can also pass plain text or numbers directly (e.g., Hello, 42, true), but using Magic Text keeps the block flexible and able to handle data provided by the macro.

    Output Parameters

    Outputs allow an Action Block to return results back to the macro. Theyâ€™re optional, but extremely helpful when you want the block to calculate something and send the result out. Think of outputs as returning results, similar to a function in programming.

    Outputs: Values returned from the Action Block to the macro.

    Outputs have defaults tooâ€”if not set explicitly in the block, the default returns.

    To create an output, you must define an output variable inside the Action Block.
    For example, following the previous example, you can create an output variable named final_price and set its value using a Set Variable action inside the block.

    Once the Action Block finishes, the macro that called it will receive the final_price output. You can then store it in any variable you wantâ€”either a global variable or a local variable, depending on your needs.

    Key Benefits

    • Reusability: Use the same block in multiple macros
    • Simplify Macros: Replace many actions with a single block
    • Easy Updates: Change logic in one place
    • Cleaner Macros: Hide complexity under the hood

    Important Limitations

    • No triggers (only actions, constraints)
    • The values of the output variables disappear after the block ends; if you need them, then save them in a corresponding variable outside the action block.
    • Cannot run independently; must be called from a macro or another block

    Differences Between Action Blocks and Macros

    Action Blocks are like functions, while macros are full programs. Here's a quick comparison:

    Capability Macros Action Blocks
    Has a disabling switch Yes No
    Can be categorized Yes Yes
    Has triggers Yes No
    Triggers can be constrained individually Yes N/A
    Has actions Yes Yes
    Actions can include â€œWait For Trigger" Yes No
    Actions can be grouped into condition sets Yes Yes
    Actions can be constrained individually Yes Yes
    Has constraints applied to all actions Yes No
    Can create/read/update global variables Yes Yes
    Can create/read/update calling macro's local variables N/A No
    Can include Input variables No Yes
    Can include Working variables No Yes
    Can include Output variables No Yes

    Example: "Send Custom Notification" Action Block

    INPUTS:
  - title (Text)
  - message (Text)

WORKING VARIABLES:
  - tempTimestamp (Text)

ACTIONS:
1. Set [tempTimestamp] = Current time
2. Create notification:
   Title = {title}
   Text = {message} at {tempTimestamp}

OUTPUTS:
  - None
